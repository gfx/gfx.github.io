71=head1 NAME
72
73Router::Simple - simple HTTP router
74
75=head1 SYNOPSIS
76
77 use Router::Simple;
78
79 my $router = Router::Simple->new();
80 $router->connect('/', {controller => 'Root', action => 'show'});
81 $router->connect('/blog/{year}/{month}', {controller => 'Blog', action => 'monthly'});
82
83 my $app = sub {
84 my $env = shift;
85 if (my $p = $router->match($env)) {
86 # $p = { controller => 'Blog', action => 'monthly', ... }
87 } else {
88 [404, [], ['not found']];
89 }
90 };
91
92=head1 DESCRIPTION
93
94Router::Simple is a simple router class.
95
96Its main purpose is to serve as a dispatcher for web applications.
97
98Router::Simple can match against PSGI C<$env> directly, which means
99it's easy to use with PSGI supporting web frameworks.
100
101=head1 HOW TO WRITE A ROUTING RULE
102
103=head2 plain string
104
105 $router->connect( '/foo', { controller => 'Root', action => 'foo' } );
106
107=head2 :name notation
108
109 $router->connect( '/wiki/:page', { controller => 'WikiPage', action => 'show' } );
110 ...
111 $router->match('/wiki/john');
112 # => {controller => 'WikiPage', action => 'show', page => 'john' }
113
114':name' notation matches qr{([^/]+)}.
115
116=head2 '*' notation
117
118 $router->connect( '/download/*.*', { controller => 'Download', action => 'file' } );
119 ...
120 $router->match('/download/path/to/file.xml');
121 # => {controller => 'Download', action => 'file', splat => ['path/to/file', 'xml'] }
122
123'*' notation matches qr{(.+)}. You will get the captured argument as
124an array ref for the special key C<splat>.
125
126=head2 '{year}' notation
127
128 $router->connect( '/blog/{year}', { controller => 'Blog', action => 'yearly' } );
129 ...
130 $router->match('/blog/2010');
131 # => {controller => 'Blog', action => 'yearly', year => 2010 }
132
133'{year}' notation matches qr{([^/]+)}, and it will be captured.
134
135=head2 '{year:[0-9]+}' notation
136
137 $router->connect( '/blog/{year:[0-9]+}/{month:[0-9]{2}}', { controller => 'Blog', action => 'monthly' } );
138 ...
139 $router->match('/blog/2010/04');
140 # => {controller => 'Blog', action => 'monthly', year => 2010, month => '04' }
141
142You can specify regular expressions in named captures.
143
144=head2 regexp
145
146 $router->connect( qr{/blog/(\d+)/([0-9]{2})', { controller => 'Blog', action => 'monthly' } );
147 ...
148 $router->match('/blog/2010/04');
149 # => {controller => 'Blog', action => 'monthly', splat => [2010, '04'] }
150
151You can use Perl5's powerful regexp directly, and the captured values
152are stored in the special key C<splat>.
153
154=head1 METHODS
155
156=over 4
157
158=item my $router = Router::Simple->new();
159
160Creates a new instance of Router::Simple.
161
162=item $router->connect([$name, ] $pattern, \%destination[, \%options])
163
164Adds a new rule to $router.
165
166 $router->connect( '/', { controller => 'Root', action => 'index' } );
167 $router->connect( 'show_entry', '/blog/:id',
168 { controller => 'Blog', action => 'show' } );
169 $router->connect( '/blog/:id', { controller => 'Blog', action => 'show' } );
170 $router->connect( '/comment', { controller => 'Comment', action => 'new_comment' }, {method => 'POST'} );
171
172C<\%destination> will be used by I<match> method.
173
174You can specify some optional things to C<\%options>. The current
175version supports 'method', 'host', and 'on_match'.
176
177=over 4
178
179=item method
180
181'method' is an ArrayRef[String] or String that matches B<REQUEST_METHOD> in $req.
182
183=item host
184
185'host' is a String or Regexp that matches B<HTTP_HOST> in $req.
186
187=item on_match
188
189 $r->connect(
190 '/{controller}/{action}/{id}',
191 {},
192 {
193 on_match => sub {
194 my($env, $match) = @_;
195 $match->{referer} = $env->{HTTP_REFERER};
196 return 1;
197 }
198 }
199 );
200
201A function that evaluates the request. Its signature must be C<<
202($environ, $match) => bool >>. It should return true if the match is
203successful or false otherwise. The first arg is C<$env> which is
204either a PSGI environment or a request path, depending on what you
205pass to C<match> method; the second is the routing variables that
206would be returned if the match succeeds.
207
208The function can modify C<$env> (in case it's a reference) and
209C<$match> in place to affect which variables are returned. This allows
210a wide range of transformations.
211
212=back
213
214=item $router->submapper($path, [\%dest, [\%opt]])
215
216 $router->submapper('/entry/', {controller => 'Entry'})
217
218This method is shorthand for creating new instance of L<Router::Simple::Submapper>.
219
220The arguments will be passed to C<< Router::Simple::SubMapper->new(%args) >>.
221
222=item $match = $router->match($env|$path)
223
224Matches a URL against one of the contained routes.
225
226The parameter is either a L<PSGI> $env or a plain string that
227represents a path.
228
229This method returns a plain hashref that would look like:
230
231 {
232 controller => 'Blog',
233 action => 'daily',
234 year => 2010, month => '03', day => '04',
235 }
236
237It returns undef if no valid match is found.
238
239=item my ($match, $route) = $router->routematch($env|$path);
240
241Match a URL against against one of the routes contained.
242
243Will return undef if no valid match is found, otherwise a
244result hashref and a L<Router::Simple::Route> object is returned.
245
246=item $router->as_string()
247
248Dumps $router as string.
249
250Example output:
251
252 home GET /
253 blog_monthly GET /blog/{year}/{month}
254 GET /blog/{year:\d{1,4}}/{month:\d{2}}/{day:\d\d}
255 POST /comment
256 GET /
257
258=back
